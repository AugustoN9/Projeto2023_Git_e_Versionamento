CICLO DE VIDA DOS ARQUIVOS

Os arquivos do seu projeto passam por diferentes estágios do git durante a criação, edição e deleção. Na imagem a seguir temos uma visão macro de como funciona o ciclo de vida de um arquivo dentro do repositório.

Explicação dos Estados:

  • Untracked (não rastreado/observado) – O arquivo foi criado mas ainda não foi adicionado ao repositório e não está sobre controle do git;
  • Edit the file (editar o arquivo) – Foi feita alguma alteração posterior à adição do arquivo e agora ele está em estado Modified;
  • Stage the file (colocar o arquivo em fila) – foi usado o comando “git add .” e agora o arquivo está em estado Staged;
  • Commit the file (salvar o estado do arquivo) – foi usado o comando “git commit” e agora o arquivo voltou ao estado Unmodified
